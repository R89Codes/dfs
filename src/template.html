Cases, Space, and Why This Matters

Use this section for CLO-1 and CLO-2: complexity, search-order impact, and the DP/topological connection.

DFS marks each vertex once and scans each adjacency list once, so its work is O(V + E). BFS does the same counting with a queue, so it is also O(V + E). What changes is the search order, the frontier structure, and the order in which timestamps, DFS trees, and topo output appear.
